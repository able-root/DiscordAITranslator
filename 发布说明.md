# DiscordAITranslator 发布说明

## 发布信息

- 插件：`DiscordAITranslator`
- 作者：`ROOT94`
- 版本：`0.3.26`
- 建议 GitHub Release 标题：`DiscordAITranslator v0.3.26 - Scroll Anchor, Manual Button Independence & Same-Language Guards`

## 版本定位

`0.3.24 - 0.3.26` 这一轮主要解决三类问题：

- 设置页滚动和交互跳动
- 同语言内容被错误交给 AI 改写
- 译文插入后消息列表滚动位置不稳定

其中 `0.3.26.scroll-anchor-manual-button-fix` 是这一轮的最新综合基线。

## 主要更新

### 0.3.24

- 修复设置页 `Select` / 输入框点击时滚动跳顶
- 减少下拉展开时的闪烁和位移

### 0.3.25

- 发送消息前增加“同语言跳过”
- 发送源语言和目标语言相同时直接发送原文
- 增加发送结果兜底，避免 AI 把原文改写成同语言句子

### 0.3.26

- 收到消息自动翻译前增加同目标语言预检测
- 手动翻译强化目标语言约束
- 移除短文本长度跳过，补足短语气词翻译
- 从自动保护包裹符中移除 `||`
- 手动翻译尽量只取当前消息正文，不把引用预览混进去
- 手动翻译后增加滚动锁
- 自动翻译刷新改为消息锚点恢复
- 单条消息翻译按钮独立于输入框总翻译开关

## 当前工作区补充修复

基于你这次反馈，当前工作区还补了一层缓存防回归：

- 旧版本已经缓存的“同语言改写译文”不会继续复用
- 如果当前消息按最新规则应当跳过自动翻译，会顺带清掉不该继续显示的旧缓存
- 缓存签名已提升，避免旧缓存直接沿用到当前基线

## 已包含的核心能力

- 新消息自动翻译
- 历史消息批量翻译
- 翻译缓存与分批处理
- mention / emoji / URL / 代码块 / 行内代码保护
- 用户自定义保护词 / 保护短语
- 自动保护包裹符规则
- 回复预览与跳转隔离
- 状态胶囊仅显示在聊天界面

## 当前重点回归点

1. 输入框总翻译开关关闭后，单条消息翻译按钮是否仍可用
2. 自动翻译插入译文或新消息进入时，页面是否还会跳到中间
3. 收到中文消息且目标语言也是中文时，是否不再生成中文改写
4. 手动翻译外语消息时，是否稳定输出设置的目标语言
5. `嘻嘻 / 哈哈 / 嗯嗯` 是否仍会被漏掉
6. `||剧透内容||` 是否不再被旧包裹符规则保护
