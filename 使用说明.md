# DiscordAITranslator 使用说明

## 1. 插件定位

`DiscordAITranslator` 是一个运行在 BetterDiscord 上的 AI 翻译插件，核心目标是：

- 让发送消息前翻译更顺手
- 让接收消息自动翻译更可控
- 让翻译规则尽量不打断原本聊天体验

## 2. 推荐的设置顺序

首次使用建议按下面顺序配置：

1. 在“翻译服务”里选择主服务商
2. 填入 API Key
3. 如有需要，再填 Endpoint 和 Model
4. 在“语言设置”里设置发送和接收语言
5. 再去调整“显示与交互”
6. 最后按需设置“高级功能”

## 3. 翻译服务说明

当前版本常用服务商包括：

- DeepSeek
- Azure Translator
- Google Cloud Translation
- DeepL
- OpenAI Compatible

如果你只是想先用起来，推荐优先试：

1. `DeepSeek`
2. `Azure Translator`
3. `Google Cloud Translation`

## 4. 语言设置

### 发送与接收语言

你可以分别设置：

- 发送消息的源语言
- 发送消息的目标语言
- 接收消息的源语言
- 接收消息的目标语言

### 发送前只翻这些源语言

这个选项适合混合聊天场景。

例如你只想“检测到英文时才翻译成中文”，就把允许翻译的源语言限制为英文。

## 5. 接收消息自动翻译

这个部分决定插件收到消息后，要不要自动替你翻译。

### 推荐模式

如果你不想太乱，优先使用：

- 只翻译新消息

如果你要回看频道历史，也想让已加载消息一起翻译，可以改成：

- 翻译当前已加载消息

### 常用过滤规则

你可以进一步控制：

- 跳过中英混合或多语言混合的消息
- 跳过本来就是目标语言的消息
- 把地区变体当成同一种语言
- 如果译文和原文几乎一样，就不显示

## 6. 显示与交互

这个部分主要影响界面表现。

### 消息显示

可以控制：

- 发送译文时是否附带原文
- 接收译文时是否同时显示原文
- 原文是否使用剧透样式
- 是否高亮译文消息
- 译文模块颜色

### 界面与按钮

可以控制：

- 输入框旁边是否显示翻译按钮
- 消息操作栏是否显示快捷翻译按钮
- 是否按频道单独记录翻译开关
- 插件界面语言

## 7. 保护规则

这是本插件非常重要的一部分。

### 保护词 / 保护短语

适合固定术语，例如：

- `BUG team`
- `ChatGPT Plus`
- `DeepSeek V3`
- `Claude 3.7 Sonnet`

这些内容会在翻译前先保护起来，翻译后再原样放回。

### 包裹符规则

适合成段保护内容，格式是：

```text
左包裹符|右包裹符
```

例如：

```text
`|`
【|】
"|"
```

### 快捷前缀

如果某些消息你根本不想翻译，可以设置跳过前缀。

例如设置：

```text
!
```

那么以 `!` 开头的消息会直接跳过翻译。

## 8. 推荐使用习惯

我更建议这样用：

1. 优先用反引号保护专有名词
2. 再用包裹符保护整段内容
3. 再补充保护词和保护短语
4. 最后再考虑跳过前缀

这样更稳，也更容易维护。

## 9. 配置校验

支持的服务商可以直接在设置面板里：

- 检测模型
- 获取模型列表

这两个按钮主要用来确认：

- API Key 是否有效
- Endpoint 是否可用
- Model 是否存在

## 10. 使用后如果没生效

如果你刚改完设置，但 Discord 里界面没立刻刷新，直接按：

```text
Ctrl+R
```

重载客户端即可。
