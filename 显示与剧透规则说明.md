# DiscordAITranslator 显示与剧透规则说明

## 1. 名词统一

- `剧透模式`
  指的是 Discord 原生的 `spoiler` 效果。
- `刮刮乐`
  是同一个效果的口语说法。

结论：
`剧透模式 = spoiler = 刮刮乐遮盖`，它们不是三个功能，只是同一个功能的不同叫法。

## 2. 四个最容易混淆的设置

### `发送译文时同时附带原文`

作用：
你自己发送消息时，把“译文 + 原文”一起发出去。

相关开关：
`发送附带原文时使用剧透/刮刮乐遮盖`

说明：
- 开启后，附带出去的原文会被 spoiler 包住，别人看到的是刮刮乐效果。
- 关闭后，附带出去的原文直接显示。

这组设置只影响“你发出去的原文”。

### `查看收到的译文时同时显示原文`

作用：
别人发来的消息被翻译后，你本地界面里是否同时显示原文。

相关开关：
`查看收到的原文时使用剧透/刮刮乐遮盖`

说明：
- 开启后，你看到的原文会以 spoiler/刮刮乐方式显示。
- 关闭后，你看到的原文直接显示。

这组设置只影响“你看到的收到消息原文”。

### `直接显示收到消息的原文`

作用：
控制收到消息的原文是单独成块显示，还是跟在译文下面显示。

说明：
- 开启：原文单独显示为一个独立块。
- 关闭：原文跟在译文后面显示。

这不是剧透开关，它只决定布局方式。

### `别人引用这条消息时显示译文+原文`

作用：
控制引用预览里是否带上“译文+原文”。

这也不是剧透开关，它只影响引用场景。

## 3. 旧字段和新字段

历史上代码里有一个旧字段：

- `useSpoilerInOriginal`

它表示“原文统一使用剧透样式”，属于早期兼容逻辑。

现在更清晰的拆分是：

- `useSpoilerInSentOriginal`
- `useSpoilerInReceivedOriginal`

也就是把剧透分成：
- 发送侧
- 接收侧

后续应继续保留旧字段兼容，但不要再把它当成新的用户功能概念来展示。

## 4. 译文颜色设置的正确预期

`译文文字颜色` 只应该影响：

- 译文颜色
- 译文强调色

不应该影响：

- 原文颜色
- Discord 默认消息文字颜色

正确效果应该是：
- 原文保持 Discord 原本颜色
- 译文使用设置中的配色

## 5. 以后判断有没有混淆，可以按这五条看

1. 写着“发送附带原文”的，只影响发送侧。
2. 写着“查看收到的原文”的，只影响接收侧。
3. 写着“剧透/刮刮乐”的，就是同一个 spoiler 效果。
4. 写着“直接显示”的，是布局，不是遮盖。
5. 写着“引用预览”的，只影响引用，不影响正文。
